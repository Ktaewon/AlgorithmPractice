# 구현(Implementation)

## 1. 정의
* 머릿 속에 있는 알고리즘을 소스코드로 바꾸는 과정
* 생각해낸 문제 풀이를 원하는 프로그래밍 언어로 정확히 구현해내야 한다
* 풀이를 떠올리는 것은 쉽지만 소스코드로 옮기기 어려운 문제
* 피지컬을 요구하는 문제

## 2. 어떤 문제가 구현하기 어려운 문제일까?
* 알고리즘은 간단한데 너무 코드가 길어지는 문제
* 특정 소수점 자리까지 출력해야 하는 문제
* 문자열이 입력으로 주어졌을 때, 한 문자 단위로 파싱해야하는 문제
* etc

## 3. 유형
* 완전 탐색
    * 모든 경우의 수를 주저 없이 다 계산하는 방법
* 시뮬레이션
    * 문제에서 제시한 알고리즘을 한 단계씩 차례대로 직접 수행

## 4. 고려해야 할 사항
1. 변수의 표현 범위

    * C/C++, Java

        * long long 이상의 문제는 거의 출제되지 X

            |정수형 종류|크기|범위|
            |---|---|---|
            |int|4 Byte|-2,147,483,648 ~ 2,147,483,647|
            |long long|8 Byte|-9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807|
            |BigInteger(Java)|가변|제한X|
    * Python
    
        * 직접 자료형 지정 X
        * 매우 큰 수의 연산 기본적으로 지원
    * 공통
        
        * 실수형 연산의 경우, 유효숫자에 따라서 원하는 결과가 나오지 않을 수 있으므로 주의!
2. 파이썬에서의 리스트 크기

    * int 자료형 데이터 개수에 따른 메모리 사용량

        |데이터 개수(리스트 길이)|메모리 사용량|
        |---|---|
        |1,000|약 4KB|
        |1,000,000|약 4MB|
        |1,000,000,000|약 40MB|
    * 데이터양이 많을 때는 꼭 메모리 제한을 고려하기!
3. 채점 환경

    * 메모리 제한, 시간 제한

        * 일반적으로 파이썬 코드가 1초에 2,000만번의 연산을 한다고 가정하고 문제를 풀면 안정적이다.

        * ex) 시간 제한 1초, 데이터 개수가 100만개의 문제가 있다면, O(NlogN) 이내의 알고리즘을 이용해야 한다.
        
            * N = 1,000,000 -> NlogN = 20,000,000
